Source C0de
Game Design Document

by Samuel Hoffmann, Max Wertheim, and Aidan Russell

Github Repository

Introduction:

This document describes the game Source C0de; a 2D side scrolling puzzle game where the player is able to interact and change the in-game physics and input systems. These mechanics will allow players to solve puzzles in fun and interactive ways not possible in other 2D puzzle games.


Technology:

Source C0de is developed on the Phaser HTML and WebGL game framework.


Gameplay:

Source C0de is a 2D platformer with unique puzzle mechanics. The player moves from level to level with a page scrolling mechanic. Each level has a unique physics puzzle to solve which provides a challenge for the player whose goal is to get to the right side of the screen. Each level has pseudo code like statements which allow the player to interact with the mechanics of the game world. The player can manipulate the code blocks by hitting, picking up, and placing them in assigned locations to make completing the levels easier.

First Level Idea: Player is shown code blocks to move forward and their keys. The player has the ability to fix the keys so they learn how to walk and also set the keys to WASD.

Final Level Idea: Fighting the final boss (pictured below), the boss opens rifts which nort has to fix one after another. After you change “boss.armored” to false you will see the source code of the boss. Lastly, you will see “boss.isAlive == true” under the source code of the boss and changing it to false will end the game.

Solve the puzzles. Save the system. Ride lightcycles. You are Nort. Win over the babes. *lightcycles will not be in the first version of the game. Nort most definitely will. The babes are up to you.

Controls:

Movement:

WASD will be used for movement and Space for jumping with various other key inputs for other actions. Most input keys will be user manipulatable allowing the user to swap keys for various actions to make beating a level easier.


Interaction:

The player will be able to interact with various aspects of the game world. The player will have the ability to hit, pick up, carry, and place various code block elements which will affect parts of the game world. By moving words or letters the player will manipulate how their character moves and how the game world acts.


Pause:

The “escape” key pauses the game!
